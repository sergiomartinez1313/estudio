🔒 Seguridad Web - Protege tus Aplicaciones

La seguridad es responsabilidad de todo desarrollador. La mayoría de brechas se deben a errores de código, no a fallos de hardware. Aprender buenas prácticas de seguridad es esencial para cualquier profesional IT.

⚠️ OWASP Top 10 - Vulnerabilidades Críticas

  1. Inyección SQL: Insertar código SQL malicioso
  2. Autenticación débil: Contraseñas y sesiones inseguras
  3. Exposición de datos sensibles: No encriptar información
  4. XML External Entities (XXE): Procesamiento inseguro de XML
  5. Control de acceso roto: Fallos en autorización
  6. Configuración incorrecta: Errores en el servidor
  7. XSS (Cross-Site Scripting): Inyectar scripts en el navegador
  8. Deserialization insegura: Objetos no confiables
  9. Componentes vulnerables: Librerías con bugs de seguridad
  10. Registro y monitoreo deficientes: No detectar ataques

💉 Inyección SQL

Vulnerable

# PELIGROSO - No hacer esto
query = f"SELECT * FROM usuarios WHERE email = '{email}'"
# Si email = "admin@example.com' --" = SELECT todo

# Entrada: ' OR '1'='1
query = f"SELECT * FROM usuarios WHERE email = '{email}'"
# Resultado: SELECT * FROM usuarios WHERE email = '' OR '1'='1'
# Obtiene todos los usuarios

Seguro

# CORRECTO - Usar prepared statements
import sqlite3

cursor = db.cursor()
query = "SELECT * FROM usuarios WHERE email = ?"
cursor.execute(query, (email,))
usuarios = cursor.fetchall()

# Flask-SQLAlchemy
usuarios = User.query.filter_by(email=email).all()

# Con ORM se previene SQL injection automáticamente

🔐 Hashing de Contraseñas

# VULNERABLE - No hacer esto import hashlib password = "mi_password" hashed = hashlib.md5(password.encode()).hexdigest() # MD5 es débil y rápido (malo para passwords)

Seguro - Usar bcrypt

import bcrypt

password = "mi_password"

# Hash
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())

# Verificar
if bcrypt.checkpw(password.encode(), hashed):
    print("Contraseña correcta")
else:
    print("Contraseña incorrecta")

# Con Django
from django.contrib.auth.hashers import make_password, check_password

hashed = make_password("mi_password")
if check_password("mi_password", hashed):
    print("Correcto")

🔑 JWT (JSON Web Tokens)

import jwt
from datetime import datetime, timedelta

SECRET_KEY = "mi_clave_secreta_muy_larga"

# Crear token
payload = {
    "user_id": 123,
    "email": "juan@example.com",
    "exp": datetime.utcnow() + timedelta(hours=1)
}

token = jwt.encode(payload, SECRET_KEY, algorithm="HS256")

# Verificar token
try:
    decoded = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
    print(decoded)
except jwt.ExpiredSignatureError:
    print("Token expirado")
except jwt.InvalidTokenError:
    print("Token inválido")

🛡️ CORS (Cross-Origin Resource Sharing)

# Express.js
const cors = require('cors');

// Permitir solo ciertos orígenes
const corsOptions = {
    origin: 'https://example.com',
    methods: ['GET', 'POST'],
    credentials: true
};

app.use(cors(corsOptions));

# Flask
from flask_cors import CORS

CORS(app, resources={
    r"/api/*": {
        "origins": ["https://example.com"],
        "methods": ["GET", "POST"]
    }
})

🔐 Buenas Prácticas de Seguridad

  • Validar siempre: Entrada del usuario es maliciosa por defecto
  • Sanitizar: Limpiar datos antes de procesar
  • Usar HTTPS: Siempre encriptar en tránsito
  • Contraseñas fuertes: Mínimo 12 caracteres, mixtas
  • Actualizaciones: Mantén librerías al día
  • Principio de mínimo privilegio: Solo los permisos necesarios
  • Logs de auditoría: Registra acceso y cambios
  • Rate limiting: Limita intentos de login
  • Variables de entorno: Nunca hardcodear secretos
  • Pruebas de seguridad: Realiza penetration tests
