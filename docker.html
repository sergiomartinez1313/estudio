🐳 Docker - Contenedorización

Docker es una plataforma de contenedorización que permite empaquetar aplicaciones con todas sus dependencias. Los contenedores garantizan que tu aplicación funcione igual en cualquier máquina, eliminando el problema "funciona en mi máquina".

🎯 ¿Por Qué Docker?

  • Consistencia: Funciona igual en desarrollo, testing y producción
  • Aislamiento: Las aplicaciones no interfieren entre sí
  • Escalabilidad: Fácil de replicar y escalar
  • Eficiencia: Más ligero que máquinas virtuales
  • DevOps: Facilita el deployment y la automatización

📦 Conceptos Clave

  • Imagen: Plantilla de solo lectura con toda la aplicación
  • Contenedor: Instancia ejecutable de una imagen
  • Dockerfile: Archivo con instrucciones para construir una imagen
  • Docker Hub: Registro público de imágenes
  • Volumen: Almacenamiento persistente para contenedores
  • Red: Conexión entre contenedores

📥 Instalación de Docker

# En Ubuntu/Debian
sudo apt update
sudo apt install docker.io
sudo apt install docker-compose

# Verificar instalación
docker --version
docker-compose --version

# Dar permisos de usuario
sudo usermod -aG docker $USER

📄 Crear un Dockerfile

# Dockerfile para aplicación Python
FROM python:3.9

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "app.py"]

🔨 Comandos Básicos de Docker

# Construir imagen
docker build -t mi-app:1.0 .

# Listar imágenes
docker images

# Ejecutar contenedor
docker run -d --name mi-contenedor mi-app:1.0
docker run -p 8000:8000 mi-app:1.0    # Con puerto
docker run -v /datos:/app/datos mi-app:1.0  # Con volumen

# Listar contenedores
docker ps              # Activos
docker ps -a           # Todos

# Ver logs
docker logs mi-contenedor
docker logs -f mi-contenedor   # En tiempo real

# Entrar en contenedor
docker exec -it mi-contenedor bash
docker exec -it mi-contenedor sh

# Parar contenedor
docker stop mi-contenedor

# Eliminar contenedor
docker rm mi-contenedor

# Eliminar imagen
docker rmi mi-app:1.0

📋 Docker Compose

# docker-compose.yml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./:/app
    environment:
      - DEBUG=True
    
  db:
    image: postgres:13
    environment:
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:

# Comandos
docker-compose up -d           # Levantar servicios
docker-compose down            # Bajar servicios
docker-compose logs -f         # Ver logs
docker-compose ps              # Ver estado

🌐 Ejemplo Completo - Aplicación Web

# app.py (Flask)
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello from Docker!'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

# requirements.txt
Flask==2.0.1

# Dockerfile
FROM python:3.9
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["python", "app.py"]

# Construir y ejecutar
docker build -t mi-app .
docker run -p 5000:5000 mi-app
