1. Introducción a AWS

¿Qué es AWS?

Amazon Web Services es una plataforma de computación en la nube que ofrece servicios bajo demanda con modelo de pago por uso.

Ventajas de AWS:

  • Escalabilidad: Crece con tu demanda automáticamente
  • Confiabilidad: SLA de 99.99% de disponibilidad
  • Costo: Paga solo por lo que usas
  • Seguridad: Múltiples capas de seguridad
  • Flexibilidad: Elige qué, cuándo y cómo desplegar

Servicios Principales

  • Compute: EC2, Lambda, ECS, Lightsail
  • Storage: S3, EBS, Glacier, FSx
  • Database: RDS, DynamoDB, Redshift
  • Networking: VPC, CloudFront, Route 53
  • Management: CloudWatch, CloudFormation, IAM

Regiones y Zonas de Disponibilidad

Regiones: Ubicaciones geográficas (us-east-1, eu-west-1, ap-southeast-1)
Zonas de Disponibilidad (AZ): Data centers independientes dentro de una región
Ejemplo: us-east-1a, us-east-1b, us-east-1c

2. EC2 - Elastic Compute Cloud

¿Qué es EC2?

Servidores virtuales (instancias) en la nube. Permiten alojar aplicaciones web, bases de datos, etc.

Tipos de Instancias

  • General Purpose (t, m): Balance CPU/RAM/Red
  • Compute Optimized (c): Alto procesamiento
  • Memory Optimized (r, x): Mucha RAM
  • Storage Optimized (i, h, d): Disco rápido
  • GPU (p, g): Aceleración GPU

Lanzar una Instancia EC2

1. Elegir AMI (Amazon Machine Image): Sistema Operativo
2. Elegir tipo de instancia: t2.micro, t2.small, etc.
3. Configurar detalles: VPC, subnet, auto-assign IP
4. Agregar storage: Volumen EBS
5. Hacer tag: Etiquetar para identificar
6. Security group: Firewall de instancia
7. Key pair: Para conectar via SSH
8. Lanzar

Conectar a EC2 vía SSH

ssh -i mi-llave.pem ubuntu@ec2-xx-xxx-xx-xxx.compute-1.amazonaws.com

# Útil: chmod 400 mi-llave.pem (permisos correctos)

# Desde Windows, usar PuTTY o WSL

Opciones de Precios EC2

  • On-Demand: Paga por hora, sin compromiso
  • Reserved Instances: 1-3 años, 40% descuento
  • Spot Instances: Hasta 90% descuento, puede terminarse
  • Dedicated Hosts: Hardware físico exclusivo

Auto Scaling

Crear/eliminar instancias automáticamente según demanda.

-- Configurar grupo de auto-scaling
1. Crear Launch Template con configuración de instancia
2. Crear Auto Scaling Group
3. Definir capacidad min/max y deseada
4. Configurar políticas de escalado:
   - Target Tracking: Mantener métrica en rango
   - Step Scaling: Agregar/quitar según condiciones

3. S3 - Simple Storage Service

¿Qué es S3?

Almacenamiento de objetos escalable infinitamente. Ideal para archivos estáticos, backups, data lakes, etc.

Conceptos Básicos

  • Bucket: Contenedor principal (nombre único globalmente)
  • Object: Archivo individual con key (ruta) y value (contenido)
  • Región: Ubicación del bucket

Crear Bucket

-- Vía AWS CLI
aws s3 mb s3://mi-bucket-unico --region us-east-1

-- Vía SDK Python (Boto3)
import boto3
s3 = boto3.client('s3')
s3.create_bucket(Bucket='mi-bucket-unico')

Operaciones Básicas S3

-- Subir archivo
aws s3 cp archivo.txt s3://mi-bucket/

-- Descargar archivo
aws s3 cp s3://mi-bucket/archivo.txt .

-- Listar contenido bucket
aws s3 ls s3://mi-bucket/

-- Eliminar archivo
aws s3 rm s3://mi-bucket/archivo.txt

-- Sincronizar carpeta local con S3
aws s3 sync ./local-folder s3://mi-bucket/

Clases de Almacenamiento S3

  • S3 Standard: Acceso frecuente, caro
  • S3 IA (Infrequent Access): Acceso infrecuente, barato
  • S3 Glacier: Archivado a largo plazo, muy barato
  • S3 Intelligent-Tiering: Mueve automáticamente entre clases

Subir Archivo desde Python

import boto3

s3 = boto3.client('s3')

# Subir archivo local
s3.upload_file(
    Filename='mi-archivo.txt',
    Bucket='mi-bucket',
    Key='carpeta/mi-archivo.txt'
)

# Generar URL presignada (acceso temporal)
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'mi-bucket', 'Key': 'mi-archivo.txt'},
    ExpiresIn=3600  # 1 hora
)

Control de Acceso

-- Hacer bucket privado (por defecto)
aws s3api put-bucket-acl --bucket mi-bucket --acl private

-- Permitir acceso público a un objeto
aws s3api put-object-acl --bucket mi-bucket --key mi-archivo.txt --acl public-read

-- Usar bucket policy para acceso específico
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::mi-bucket/*"
  }]
}

4. RDS - Relational Database Service

¿Qué es RDS?

Servicio de base de datos administrado. AWS se encarga de backups, parches, replicación.

Motores Soportados

  • MySQL / MariaDB
  • PostgreSQL
  • Oracle
  • SQL Server
  • Amazon Aurora

Crear Instancia RDS

-- Vía AWS CLI
aws rds create-db-instance \
  --db-instance-identifier mi-bd-mysql \
  --db-instance-class db.t2.micro \
  --engine mysql \
  --master-username admin \
  --master-user-password MinhaContraseña123 \
  --allocated-storage 20

Multi-AZ Deployment

Replica síncrona en otra AZ para alta disponibilidad.

-- RDS Standard (Single-AZ)
Costo menor, menos disponibilidad

-- RDS Multi-AZ
Costo mayor, failover automático en segundos
Replicación síncrona a otra AZ

Backup y Restore

-- RDS realiza backups automáticos:
- Daily snapshots (retención: 1-35 días)
- Transaction logs para point-in-time restore

-- Restaurar a un punto en tiempo
aws rds restore-db-instance-to-point-in-time \
  --source-db-instance-identifier mi-bd-original \
  --target-db-instance-identifier mi-bd-restaurada \
  --restore-time 2024-03-20T14:30:00Z

Read Replicas

Copias de lectura para distribuir carga de lectura.

-- Crear read replica en otra región
aws rds create-db-instance-read-replica \
  --db-instance-identifier mi-bd-replica \
  --source-db-instance-identifier mi-bd-principal \
  --db-instance-class db.t2.small

5. Lambda - Serverless Computing

¿Qué es Lambda?

Ejecutar código sin servidores. Paga solo por tiempo de ejecución, sin pensar en infraestructura.

Lenguajes Soportados

  • Python
  • Node.js
  • Java
  • C# (.NET)
  • Go
  • Ruby
  • Custom runtimes

Crear Función Lambda (Python)

def lambda_handler(event, context):
    print(f"Evento recibido: {event}")
    
    nombre = event.get('nombre', 'Mundo')
    
    return {
        'statusCode': 200,
        'body': f'Hola {nombre}!'
    }

Disparadores (Triggers) Lambda

  • API Gateway: Invocación vía HTTP
  • S3: Al subir archivo
  • DynamoDB: Cambio en tabla
  • CloudWatch: Según horario (cron)
  • SNS/SQS: Mensaje de cola
  • EventBridge: Eventos personalizados

Ejemplo: Lambda con API Gateway

def lambda_handler(event, context):
    # event contiene: path, httpMethod, headers, body, etc.
    
    if event['httpMethod'] == 'GET':
        return {
            'statusCode': 200,
            'body': json.dumps({'mensaje': 'GET realizó'})
        }
    
    elif event['httpMethod'] == 'POST':
        body = json.loads(event['body'])
        return {
            'statusCode': 201,
            'body': json.dumps({'id': 123, 'datos': body})
        }

Limitaciones Lambda

  • Timeout máximo: 15 minutos
  • Memoria: 128 MB - 10 GB
  • Tamaño de archivo: 250 MB (descomprimido)
  • Almacenamiento temporal: 512 MB

Layers Lambda

Compartir código/dependencias entre funciones.

-- Crear layer con dependencias Python
mkdir python
pip install requests -t python/
zip -r mi-layer.zip python/

-- Subir via AWS CLI
aws lambda publish-layer-version \
  --layer-name mis-dependencias \
  --zip-file fileb://mi-layer.zip \
  --compatible-runtimes python3.9

6. Networking - VPC y Conectividad

VPC (Virtual Private Cloud)

Red privada en AWS donde residen tus recursos.

Componentes VPC

  • Subnets: Rango de IPs dentro de VPC
  • Internet Gateway: Conecta VPC a internet
  • Route Tables: Reglas de enrutamiento
  • Security Groups: Firewall de instancia
  • Network ACLs: Firewall de subred

Security Groups

Controla tráfico de entrada y salida.

-- Regla típica: Permitir SSH desde tu IP
Inbound: TCP puerto 22, fuente XX.XX.XX.XX/32

-- Regla: Permitir HTTP/HTTPS público
Inbound: TCP puerto 80, fuente 0.0.0.0/0
Inbound: TCP puerto 443, fuente 0.0.0.0/0

-- Salida por defecto
Outbound: Todo permitido por defecto

Conectividad

  • Internet: Via Internet Gateway + Route Table
  • On-premises: Via VPN (Site-to-Site VPN)
  • Entre VPCs: VPC Peering o Transit Gateway

Elastic IP

Dirección IP pública estática (no cambia al reiniciar).

-- Asignar Elastic IP a instancia
aws ec2 allocate-address --domain vpc
aws ec2 associate-address --instance-id i-xxxxx --allocation-id eipalloc-xxxxx

CloudFront - CDN

Distribuye contenido globalmente con baja latencia.

Origen: S3 bucket o servidor custom
Edge Locations: Cachés en 300+ ubicaciones mundiales
Result: Usuarios acceden contenido desde ubicación más cercana

IAM - Gestión de Acceso

Conceptos Básicos IAM

  • Usuarios: Identidades individuales
  • Grupos: Conjunto de usuarios
  • Roles: Conjuntos de permisos reutilizables
  • Políticas: Reglas de permiso (JSON)

Crear Usuario IAM

aws iam create-user --user-name developer1
aws iam create-access-key --user-name developer1
aws iam attach-user-policy --user-name developer1 --policy-arn arn:aws:iam::aws:policy/AmazonEC2FullAccess

Monitoreo y Logs

CloudWatch

Servicio de monitoreo centralizado.

-- Métricas: CPU, Red, Disco
-- Logs: De aplicaciones y servicios
-- Alarmas: Notificaciones cuando métrica cruza umbral
-- Dashboard: Visualización personalizada
