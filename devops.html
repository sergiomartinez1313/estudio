⚙️ DevOps - Deployment y Automatización

DevOps es la combinación de Development y Operations. Se enfoca en automatizar el deployment, monitoreo y mantenimiento de aplicaciones. Los desarrolladores modernos necesitan entender deployment y operaciones.

🎯 ¿Qué es DevOps?

  • Automatización: Automatizar tareas repetitivas
  • Continuidad: Continuous Integration y Continuous Deployment
  • Monitoreo: Vigilar aplicaciones en producción
  • Escalabilidad: Crecer sin interrupciones
  • Confiabilidad: Sistemas resilientes y tolerantes a fallos
  • Colaboración: Developers y Operations trabajando juntos

🔄 CI/CD (Continuous Integration / Continuous Deployment)

Continuous Integration (CI)

# Proceso
1. Developer hace commit
2. Sistema automáticamente:
   - Descarga código
   - Ejecuta tests
   - Verifica estilos de código
   - Genera reportes

3. Si falla algo, developer lo corrige
4. Si todo ok, pasa a siguiente etapa

Continuous Deployment (CD)

# Proceso
1. Código pasa CI
2. Automáticamente se:
   - Construye imagen Docker
   - Se despliega a staging
   - Se ejecutan tests E2E
   - Se despliega a producción

3. Sin intervención manual

💻 GitHub Actions

# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v2
    
    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.9'
    
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
    
    - name: Run tests
      run: pytest
    
    - name: Build Docker image
      run: docker build -t mi-app:latest .
    
    - name: Push to Docker Hub
      run: |
        docker login -u ${{ secrets.DOCKER_USER }} -p ${{ secrets.DOCKER_PASS }}
        docker push mi-app:latest

  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
    - name: Deploy to production
      run: |
        ssh -i ${{ secrets.SSH_KEY }} user@server "docker pull mi-app:latest && docker run -d -p 80:5000 mi-app:latest"

📊 Monitoreo y Logs

Prometheus + Grafana

# prometheus.yml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'mi-app'
    static_configs:
      - targets: ['localhost:9090']

# docker-compose.yml
version: '3'
services:
  prometheus:
    image: prom/prometheus:latest
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
  
  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"

Heroku

# Procfile
web: gunicorn app:app

# Commands
heroku create mi-app
git push heroku main
heroku logs --tail

# Escalar
heroku ps:scale web=2

Kubernetes

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mi-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mi-app
  template:
    metadata:
      labels:
        app: mi-app
    spec:
      containers:
      - name: mi-app
        image: mi-app:latest
        ports:
        - containerPort: 5000
        resources:
          limits:
            memory: "512Mi"
            cpu: "500m"

# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: mi-app-service
spec:
  selector:
    app: mi-app
  ports:
    - protocol: TCP
      port: 80
      targetPort: 5000
  type: LoadBalancer

# Desplegar
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
kubectl get pods

🔍 Mejores Prácticas DevOps

  • Infrastructure as Code: Define infraestructura con código
  • Automatización: Automatiza todo lo posible
  • Versionado: Versiona todo incluyendo configuración
  • Testing: Unit tests, integration tests, E2E tests
  • Logs centralizados: Recopila logs en un lugar
  • Monitoreo activo: Alertas antes de problemas
  • Rollback rápido: Poder volver atrás si falla
  • Feature flags: Activar/desactivar features sin deploy
  • Blue-green deployment: Cambiar sin downtime
  • Disaster recovery: Plan de recuperación ante crisis
