🚀 GitHub - Control de Versiones Profesional

GitHub es la plataforma líder para alojar repositorios Git. Te permite colaborar con otros desarrolladores, controlar versiones de código y mantener un historial completo de cambios.

📌 Conceptos Fundamentales de Git

  • Repositorio (Repo): Carpeta del proyecto con control de versiones
  • Commit: Foto/snapshot de los cambios en un momento específico
  • Branch: Versión paralela del proyecto para trabajo independiente
  • Merge: Juntar cambios de una rama a otra
  • Push: Subir cambios locales a GitHub
  • Pull: Traer cambios de GitHub a tu máquina local
  • Clone: Descargar un repositorio completo de GitHub
  • Fork: Copia personal de un proyecto de otro usuario
  • Pull Request: Solicitud para fusionar cambios con revisión

🔧 Configuración Inicial de Git

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
git config --list              # Ver configuración

📁 Crear y Clonar Repositorios

Inicializar un nuevo repo local:

mkdir mi-proyecto
cd mi-proyecto
git init                       # Inicia repositorio
git status                     # Ver estado

Clonar un repositorio existente:

git clone https://github.com/usuario/proyecto.git
cd proyecto

💾 Flujo Básico de Trabajo (Local)

# 1. Crear/modificar archivos
echo "contenido" > archivo.txt

# 2. Ver cambios
git status
git diff                       # Ver diferencias

# 3. Preparar cambios (staging)
git add archivo.txt            # Un archivo
git add .                      # Todos los archivos

# 4. Confirmar cambios (commit)
git commit -m "Mensaje descriptivo"

# 5. Ver historial
git log
git log --oneline              # Versión corta

🌐 Conectar con GitHub y Subir Cambios

# 1. Crear repositorio en GitHub
# 2. Agregar URL remota
git remote add origin https://github.com/tuusuario/mi-proyecto.git
git remote -v                  # Ver remoto configurado

# 3. Primera vez subiendo
git branch -M main             # Renombrar a main
git push -u origin main        # Subir a GitHub

# 4. Siguientes cambios
git push                       # Solo push
git push origin main           # Especificar rama

🔀 Trabajar con Ramas

# Ver ramas
git branch                     # Ramas locales
git branch -a                  # Todas las ramas

# Crear rama
git branch nueva-funcionalidad
git checkout -b nueva-rama     # Crear y cambiar

# Cambiar de rama
git checkout main
git checkout nombre-rama

# Eliminar rama
git branch -d rama-vieja       # Local
git push origin --delete rama  # Remoto

# Fusionar ramas
git checkout main
git merge nueva-funcionalidad

🔄 Actualizar desde GitHub

# Traer cambios
git fetch                      # Descargar cambios
git pull                       # Descargar y fusionar
git pull origin main           # Rama específica

🆘 Deshaciendo Cambios

# Deshacer cambios locales
git checkout -- archivo.txt    # Deshacer archivo
git reset HEAD archivo.txt     # Sacar del staging

# Modificar último commit
git commit --amend -m "Nuevo mensaje"

# Deshacer commit
git revert HEAD                # Crear nuevo commit que deshace
git reset HEAD~1               # Volver atrás (cuidado!)

👥 Colaboración y Pull Requests

# Workflow colaborativo:
# 1. Fork el proyecto
# 2. Clone tu fork
git clone https://github.com/TU_USUARIO/proyecto.git

# 3. Crear rama para tu cambio
git checkout -b fix/bug-importante

# 4. Hacer cambios y commit
git add .
git commit -m "Arreglado: descripción del bug"

# 5. Push a tu fork
git push origin fix/bug-importante

# 6. Crear Pull Request en GitHub
# 7. Esperar revisión y merge

📊 Ver Historial y Cambios

git log                        # Historial completo
git log --oneline -10          # Últimos 10 commits
git log --author="nombre"      # Commits de un autor
git log -p                     # Ver cambios detallados
git diff HEAD~1                # Diferencia con commit anterior
git show commit-id             # Ver cambios de un commit

⚡ Comandos Útiles Adicionales

git status                     # Estado actual
git add --interactive          # Seleccionar cambios
git stash                      # Guardar cambios temporales
git stash pop                  # Recuperar cambios
git tag v1.0.0                 # Crear etiqueta
git push --tags                # Subir etiquetas

🎯 Buenas Prácticas

  • Hacer commits pequeños y frecuentes con mensajes claros
  • Usar ramas para nuevas funcionalidades
  • Revisar cambios antes de commit (git diff)
  • Hacer Pull antes de Push para evitar conflictos
  • Dar nombres descriptivos a ramas y commits
  • Usar .gitignore para archivos que no quieres versionar
  • Escribir mensajes de commit en presente y descriptivos
  • Hacer merge de forma regular para evitar conflictos grandes
